## 1. 项目简介

Skill Center 是公司内部的 Agent 技能注册中心，用于发布、发现和复用 AI Agent 技能包。

**核心能力：**

- 通过 Web 界面一键发布技能
- 关键词搜索和浏览发现技能
- 命名空间和权限管理，控制技能可见性
- 自动安全扫描，检测潜在风险

**平台地址：** https://skill-center.sigmob.com

## 3. 发布技能

### 3.1 准备技能包

技能包根目录必须包含 `SKILL.md` 文件：

```markdown
---
name: my-skill
description: 技能描述
version: 1.0.0
---

# 技能说明

详细的使用文档...
```

**限制：** 单文件 ≤ 10MB，总包 ≤ 100MB，文件数 ≤ 500

### 3.2 发布步骤

1. 登录后访问 https://skill-center.sigmob.com/dashboard/publish
2. 选择**命名空间**
3. 选择**可见性**：
   - **Public** — 所有人可见
   - **Namespace Only** — 仅命名空间成员可见
   - **Private** — 仅自己可见
4. 上传 zip 文件或技能目录
5. 点击「发布」

发布后系统会自动进行安全扫描（详见第 6 章）。普通用户首次发布需等待管理员审核。

### 3.3 版本管理

- 使用语义化版本号：`major.minor.patch`（如 `1.2.3`）
- 已发布版本不可修改，需发布新版本
- `latest` 标签自动指向最新稳定版本

## 4. 搜索与浏览技能

### 搜索技能

## 5. 命名空间管理

### 概念

命名空间是技能的逻辑分组，坐标格式为 `@{namespace}/{skill-slug}`。

- `@global/my-skill` — 平台公共空间
- `@my-team/my-skill` — 团队空间

### 创建命名空间

登录后访问「我的命名空间」页面，点击「创建命名空间」，填写名称和 Slug（小写字母/数字/连字符）。

### 成员与角色

| 角色 | 权限 |
|-----|------|
| Owner | 完全控制，包括删除命名空间 |
| Admin | 管理成员、发布和审核技能 |
| Member | 发布技能、查看命名空间内技能 |

在命名空间详情页的「成员」标签中添加/管理成员。

## 6. 平台角色与权限

### 6.1 平台角色说明

平台角色是全局权限，控制用户在平台级别的操作能力。

| 角色 | 权限范围 | 适用岗位 |
|-----|---------|---------|
| **超级管理员** | 平台完全控制权，包括管理其他管理员、系统配置、所有命名空间 | 平台技术负责人、运维总监 |
| **审核员** | 技能审核与管理，可审核所有技能、隐藏/恢复技能、查看安全报告 | 技能委员会成员、技术审核负责人 |
| **用户管理员** | 用户审批与账号管理，可审批新用户、禁用/启用账号、管理用户角色 | HR、部门行政、团队 Leader |
| **审计员** | 只读审计日志与监控，可查看操作日志、安全报告，不能修改任何数据 | 安全合规团队、质量管理人员 |
| **普通用户** | 默认角色，可发布技能、搜索安装、创建命名空间 | 所有研发人员、产品经理、测试人员 |

### 6.2 角色分配建议

**技术团队：**

| 岗位 | 建议角色 | 说明 |
|-----|---------|------|
| 技术总监/架构师 | 超级管理员 | 需要管理整个平台配置 |
| 技术 Leader | 审核员 | 负责技能审核和质量把控 |
| 高级工程师 | 普通用户 | 发布和管理自己的技能 |
| 普通工程师 | 普通用户 | 发布和使用技能 |

**产品与运营团队：**

| 岗位 | 建议角色 | 说明 |
|-----|---------|------|
| 产品经理 | 普通用户 | 浏览和使用技能 |
| 运营人员 | 普通用户 | 浏览和使用技能 |

**管理与行政团队：**

| 岗位 | 建议角色 | 说明 |
|-----|---------|------|
| 部门经理 | 用户管理员 | 管理部门人员账号 |
| HR/行政 | 用户管理员 | 审批新用户、管理账号 |

**安全与合规团队：**

| 岗位 | 建议角色 | 说明 |
|-----|---------|------|
| 安全工程师 | 审计员 | 查看审计日志和安全报告 |
| 合规专员 | 审计员 | 监控操作合规性 |

### 6.3 权限层级

```
超级管理员（最高权限）
    ↓
审核员 ≈ 用户管理员（平级，各自专注不同领域）
    ↓
审计员（只读监控）
    ↓
普通用户（基础权限）
```

**重要原则：**

- 高权限角色可以管理低权限角色的用户
- 低权限角色不能操作高权限账号
- 超级管理员只能由现有超级管理员指定
- 每个用户默认分配普通用户角色

### 6.4 命名空间角色

命名空间角色是局部权限，仅在特定命名空间内生效。

| 角色 | 权限 | 适用场景 |
|-----|------|---------|
| **Owner** | 完全控制，包括删除命名空间、转移所有权 | 命名空间创建者、团队负责人 |
| **Admin** | 管理成员、发布和审核技能、修改设置 | 团队 Leader、技术负责人 |
| **Member** | 发布技能、查看命名空间内技能 | 团队成员、普通开发者 |

**命名空间角色分配建议：**

| 场景 | Owner | Admin | Member |
|-----|-------|-------|--------|
| 部门公共空间 | 部门经理 | 技术 Leader | 部门成员 |
| 项目组空间 | 项目负责人 | 核心开发 | 项目成员 |
| 个人空间 | 创建者 | - | - |

## 7. 安全扫描

### 7.1 扫描机制

所有发布的技能包都会自动经过 Skill Scanner 安全扫描，检测潜在的安全风险和恶意代码。

### 7.2 扫描内容

**敏感信息检测：**

- API Key、Secret Key、Access Token 泄露
- 数据库连接字符串、密码
- 私钥文件（SSH、SSL 证书等）
- 内部域名、IP 地址暴露

**恶意代码检测：**

- 危险命令执行（`rm -rf`、`format`、`del` 等）
- 网络请求异常（外传数据、反向 Shell）
- 文件系统破坏性操作
- 环境变量窃取

**代码质量检查：**

- 不安全的函数调用
- 未经验证的用户输入
- 硬编码的敏感信息
- 不安全的依赖项

**文件类型验证：**

- 文件扩展名白名单检查（50+ 种允许的文件类型）
- 文件签名验证（如 PNG 魔术字节、SVG 内容检查）
- 可执行文件检测

**策略规则：**

- 自定义 YARA 规则匹配
- 签名模式匹配
- 正则表达式规则

### 7.3 扫描结果

扫描完成后，结果会显示在技能详情页的「安全审计」标签：

- **通过** — 未发现安全问题，技能正常发布
- **警告** — 发现潜在风险，显示警告信息，可选择继续发布
- **失败** — 发现严重安全问题，需要修复后重新发布

### 7.4 查看扫描报告

1. 进入技能详情页
2. 切换到「安全审计」或「Security」标签
3. 查看详细的扫描结果：
   - 检测到的风险项列表
   - 风险等级（高/中/低）
   - 具体文件位置和行号
   - 修复建议

## 8. 常见问题

### 登录相关

**Q: 飞书登录后没有自动创建账号？**
请检查飞书 App 是否已授权。如仍有问题，请联系管理员。

**Q: 忘记密码？**
登录页点击「忘记密码」，通过注册邮箱重置。

### 发布相关

**Q: 发布失败提示 "SKILL.md not found"？**
确保技能包根目录包含 `SKILL.md` 文件，且包含 YAML frontmatter（`name`、`description` 字段）。

**Q: 发布失败提示 "Version already exists"？**
该版本号已发布过，请在 `SKILL.md` 中更新 `version` 字段后重新发布。

**Q: 发布后状态是 "PENDING_REVIEW"？**
普通用户首次发布需要管理员审核，审核通过后自动发布。

**Q: 安全扫描失败怎么办？**
查看技能详情页的「安全审计」标签，根据扫描报告中的修复建议修改后重新发布。

### 搜索相关

**Q: 搜索不到某些技能？**
可能是技能的可见性设置为「Namespace Only」或「Private」。请联系技能作者请求访问权限。

### 命名空间相关

**Q: 如何创建命名空间？**
登录后进入「我的命名空间」页面，点击「创建命名空间」按钮。

**Q: 如何邀请成员加入命名空间？**
在命名空间详情页的「成员」标签中，点击「添加成员」按钮。

**Q: 为什么我看不到某些命名空间的技能？**
可能是技能可见性限制。请联系命名空间管理员请求访问权限。
